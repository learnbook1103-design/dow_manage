## ERP vs Hub 역할 경계 — 필독

다우밸브는 **더존 ERP**를 운영 시스템으로 사용합니다.

```
Hub 담당 (ERP 앞단 + 지식 관리)          ERP 담당 (확정 거래 기록)
────────────────────────────────         ──────────────────────────
리드 → 미팅 → 견적 → 협의                수주확정 → 발주 → 입출고
영업 딜 추적 (deals.md)                   세금계산서 / 매출 / 재고
미팅 기록 (inbox/)                        거래처 마스터 (공식 등록)
팀별 과제 (tasks/)                        회계 / 정산
지식·온톨로지
```

**원칙: 더존에 있거나 더존에 들어가야 하는 데이터는 Hub에 넣지 않는다.**
- 발주 / 재고 / 입출고 → ERP 조회
- 수주 금액 / 세금계산서 → ERP 조회
- AI가 이 데이터를 물어보면: "해당 정보는 더존 ERP에서 확인하세요" 라고 안내할 것

## 폴더 구조

```
hub-data/
├── CLAUDE.md              ← AI 컨텍스트 (이 파일)
│
├── ontology/              ← 회사 언어 정의 (개념·명사)
│   ├── glossary.md        용어 사전
│   ├── processes.md       업무 흐름
│   └── org.md             조직 구조
│
├── companies/             ← B2B 거래처 관리
│   ├── _index.md          거래처 통합 인덱스 (명칭 정규화)
│   ├── customers/         고객사
│   │   └── [회사명]/
│   │       ├── profile.md
│   │       └── contacts/
│   └── suppliers/         공급사
│       └── _index.md
│
├── sales/                 ← 영업 (ERP 앞단)
│   ├── tasks/             팀별 과제 (팀명.md × 7)
│   ├── weekly-reports/    주간 리포트 (YYYY-MM-DD/팀명.md)
│   ├── channels/          채널별 현황
│   └── analysis/          클레임 등 분석
│
│   ※ 발주·재고·입출고는 더존 ERP에서 관리 → Hub 없음
│
├── team/                  ← 내부 조직
│   ├── members.md
│   └── okr-2026.md
│
├── project/               ← 내부 프로젝트
│   ├── ax-web/
│   ├── approval-system/
│   └── flex-hr/
│
└── inbox/                 ← 미팅·활동 인박스
```

## 영업 단계 정의

| 단계 | 설명 |
|------|------|
| 리드 | 잠재 고객 발굴 / 인바운드 문의 |
| 견적요청 | 고객이 견적 요청한 상태 |
| 견적발송 | 견적서 발송 완료, 회신 대기 |
| 협의중 | 가격·납기·사양 협의 진행 중 |
| 수주확정 | PO 또는 발주 확정 |
| 생산/납기 | 생산 중 또는 납기 조율 |
| 출고완료 | 출고 처리 완료 |
| 종료 | 수주 실패 / 계약 종료 |
